# Shoulders of Giants

> An architect-grade protocol for reusing existing code — licensed MIT, © 2026 CeaserZhao.
> Part of the **Fundemetal** family of cognitive protocols.

## One line

Reuse code as a **sovereignty decision**, not a search task.

## The problem it attacks

Most "help me find a library" requests collapse into worker mode: search keywords →
sort by stars → paste the top three repos. That skips the actual architectural decision.
Every dependency is simultaneously an asset *and* a liability — `left-pad` and the `xz`
backdoor are two reminders of what that costs.

## How it works

A strict six-stage protocol (search is stage 3, not stage 1):

1. **Sovereignty Map** — partition the project (core / supporting / generic) before any search.
2. **Problem Shape** — define the *concept*, not the keyword.
3. **Prospecting** — search only now, across many sources.
4. **Due Diligence** — evaluate on five axes; conceptual fit is a veto, stars are never the first axis.
5. **Reuse Depth** — reference, encapsulate (anti-corruption layer), fork, or extract the essence.
6. **Systems Review** — the *group photo* of all giants, checked for coherence and stacked risk.

Five hard gates stand between you and a recommendation: sovereignty declaration first,
problem-derived queries, conceptual-fit veto, mandatory escape plans, and a deliverable
that must include a decision table **plus at least one rejected candidate**.

## Why it belongs in Fundemetal

It is a *cognitive protocol*, not a tool: it forces a specific thinking discipline
(architect, not worker) with anti-cheat hard gates and explicit academic grounding
(Wardley Mapping, DDD, Hyrum's Law, the Lindy effect, supply-chain security). Plain
Markdown, MIT, drop-in.
