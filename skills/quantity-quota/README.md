# Quantity Quota

> Use when generating options, ideas, solutions, or approaches — before evaluating any of them. Forces a quota of genuinely distinct candidates and bans premature judgment. Trigger whenever the first workable idea is starting to monopolize attention.

Premature convergence: the moment a workable option appears, it anchors us, evaluation kicks in early, and the search stops — so we never reach the candidates that only show up after the obvious ones are exhausted. Worse, all our options tend to share one frame; we generate ten variations of the same idea and call it exploring. Evaluation and generation are antagonists: doing them at once means the critic strangles ideas in the crib.

## When to run
- Any "what are our options / ideas / approaches" moment.
- The instant you feel "well, obviously we'd just do X".
- Before any option gets evaluated, chosen, or defended.

## The protocol
A — Set the quota · B — Generate under silence · C — Break the frame · D — Only now, evaluate.

## Evidence
> "We need a name for our new API product. Give me your top 3 recommendations."

*Without the protocol:* produces 3 polished names with rationale — evaluation from the first token, quota of 3, single frame.
*With the protocol:* refuses to jump to 3, generates 30 names spanning multiple frames (descriptive, abstract, metaphor, invented, anti-name), keeps 21–30 in a frame the first 10 never touched, and only *then* narrows — because the good name is statistically unlikely to be in the first 3 you'd think of.
