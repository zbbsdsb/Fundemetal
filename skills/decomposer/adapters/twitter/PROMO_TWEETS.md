**Tweet 3**
```
I built Decomposer to fix this.

```

**Tweet 4**
```
Step 1 — Honesty Fuse 🚨
Before any plan, state 3 things:
• What you can actually do
• Where you're blind
• How this most likely dies
Can't write all three? You're not ready to plan. You're ready to lie.
```

**Tweet 5**
```
Step 2 — Uncertainty Map 🗺️
Tag every element:
🟢 Green = known
🟡 Yellow = known-unknown
🔴 Red = unknown-unknown
Rule: Green can't exceed 70%. All-green = you're lying to yourself.
```

**Tweet 6**
```
Step 3 — Decomposition 🌳
Not "what are the parts?" (functional).
But "what must I know before I can know X?" (cognitive).
That reveals the REAL critical path — not the one in your Gantt chart.
```

**Tweet 7**
```
Step 4 — Error Budget 📊
For every node, write: Confidence, Survival Condition, Lethality.
"Alive if 50 users sign up week 1. Dead if >10% login fails."
No node survives without a testable condition. That's the anti-shell.
```

**Tweet 8**
```
Step 5 — Anti-Shell Check 🔍
5 hard checks:
1. Every node has a survival condition?
2. Any jargon?
3. Yellow+Red ≥ 30%?
4. An explicit "I don't know"?
5. First step in one sentence?
Fail one → fix it.
```

**Tweet 9**

```
Standard AI: "We need a recommendation engine."
Decomposer: "Recommendation engine — YELLOW. I don't know if filtering works. Need 100 users of data to test."
One is a statement. The other is a testable bet.
```
